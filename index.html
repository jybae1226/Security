1. Security Architecture & Mechanisms

High-level overview of how 4G (LTE/EPC) and 5G (NR/5GC) implement security on different layers: user authentication, key management, confidentiality, integrity, and network slicing.

1.1 4G (LTE) Security

  • EPS-AKA authentication between UE, eNodeB, and EPC.
  • Key hierarchy: K, KASME, KeNB, etc.
  • Encryption and integrity algorithms (e.g., AES-based, SNOW 3G).
  • Protection of control plane vs. user plane traffic.

1.2 5G Security Enhancements

  • 5G-AKA, SUCI/SUPI and subscriber identity protection.
  • Service-based architecture (SBA) and network function security (TLS, OAuth2, etc.).
  • Roaming security improvements and home network control.
  • Network slicing and isolation assumptions.

2. Known & Possible Attacks

This section lists attacks found in the literature against 4G/5G networks. For each attack, the goal, required capabilities, and affected generation (4G, 5G, or both) are described.

2.1 Signaling & Protocol Attacks

  • Downgrade attacks (forcing a device from 4G/5G to 2G/3G).
  • Attach/TAU procedure abuse for tracking and denial of service.
  • Fake base station (IMSI catcher / stingray-like devices).

2.2 User Privacy & Location Tracking

  • Exploiting temporary identifiers (GUTI, 5G-GUTI) for long-term tracking.
  • Paging message analysis to infer presence/location of a subscriber.

2.3 Core Network & SBA Attacks (Mainly 5G)

  • Exploiting misconfigured service-based interfaces (HTTP/2, JSON APIs).
  • Lack of isolation between network slices or tenants.

3.1 Evaluation Criteria

  • Required capabilities: attacker position, access to network, RF equipment.
  • Complexity: protocol knowledge, software skills, reverse engineering.
  • Detectability: can operator detect / correlate such activity?
  • Mitigations: standard-based vs. deployment-specific countermeasures.
