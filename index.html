Security in 4G and 5G Mobile Networks

Description of security mechanisms, attacks, and feasibility analysis

This project investigates how 4G and 5G mobile networks are protected, what kinds of attacks are still possible against these systems, and how realistic such attacks are in practice in terms of required skills, cost, and impact.

Author: [JongYeon Bae] Course: [Fundamentals of the Security in Communications]

1. Introduction

Mobile networks have become the main communication infrastructure for billions of users. 4G/LTE enabled mobile broadband and always-on connectivity, while 5G promises ultra-low latency, massive machine-type communication, and support for critical services such as industrial control or connected vehicles. As a result, the security of these networks is directly linked to user privacy, safety, and even national security.

The goal of this project is to provide a clear description of the security mechanisms that protect 4G and 5G networks, to summarize known types of attacks against these systems, and to analyze the feasibility and complexity of these attacks from the perspective of a realistic attacker. The project is based on existing standards and research papers rather than on original experiments in an operator network.

2. Current State of the Art

4G/LTE and 5G are specified by the 3rd Generation Partnership Project (3GPP), which defines detailed security architectures for authentication, key management, and protection of both control-plane and user-plane traffic. Over the last decade, these specifications have evolved to address weaknesses discovered in earlier generations such as 2G and 3G, where encryption was optional and mutual authentication was often missing.

In 4G, the EPS-AKA protocol, a stronger key hierarchy, and mandatory integrity protection already delivered substantial improvements over previous systems. 5G further refines this model by introducing subscriber identity protection on the radio interface, a service-based core network architecture secured with modern web technologies (TLS, OAuth2), and more explicit handling of roaming and inter-operator trust.

At the same time, academic research has shown that even modern systems contain design and implementation flaws. Examples include protocol downgrades to weaker generations, fake base stations that exploit residual weaknesses in the attachment procedures, side-channel based location tracking, and misconfigurations of 5G service-based interfaces. Therefore, the state of the art in mobile network security is a balance between strong standardized mechanisms and practical vulnerabilities in deployments.

3. Security Architecture & Mechanisms

This section briefly summarizes how 4G and 5G secure subscriber identity, authenticate devices and networks, derive cryptographic keys, and protect signaling and user data.

3.1 4G (LTE) Security

  • Authentication (EPS-AKA): mutual authentication between the User Equipment (UE), the serving base station (eNodeB), and the core network (EPC) using long-term credentials stored in the USIM and in the operator's Home Subscriber Server (HSS).
  • Key hierarchy: a master key K is used to derive KASME and then further keys such as KeNB for the radio interface. This limits the impact of key compromise and supports key refresh during mobility.
  • Confidentiality and integrity: standardized algorithms (for example, AES or SNOW 3G based) provide encryption and integrity protection for control-plane signaling and, optionally, for user-plane data.
  • Separation of planes: different keys and security associations for control and user plane, so that signaling remains protected even if user data is handled by intermediate gateways and tunnels.

3.2 5G Security Enhancements

  • 5G-AKA and subscriber identity protection: the permanent subscriber identifier (SUPI) is never sent in clear over the air. Instead, a concealed identifier (SUCI) is transmitted, protecting users from passive IMSI-catching style attacks.
  • Service-Based Architecture (SBA): core network functions communicate via HTTP/2 and JSON APIs secured with TLS and modern authentication/authorization mechanisms, aligning mobile network security with common IT security practices.
  • Roaming and home control: the home network has more control over authentication decisions and can better enforce its policies even when the subscriber is served by a visited network.
  • Network slicing: logical networks (slices) are isolated and can be configured with different security policies depending on the service (e.g., massive IoT vs. ultra-reliable low-latency communication).

4. Known Attacks and Feasibility Analysis

Even with strong standardized security, practical attacks against 4G and 5G networks remain possible. This section describes representative attacks and evaluates their feasibility in terms of attacker capabilities, technical complexity, and required budget.

4.1 Signaling and Protocol-Level Attacks

  • Downgrade attacks: an attacker operates a fake base station that forces devices to connect using weaker generations (2G/3G) where encryption or mutual authentication may be absent, enabling eavesdropping or impersonation.
  • Attach and tracking area update abuse: by manipulating attach/TAU procedures, an attacker can cause denial of service for specific subscribers or infer their presence in a given area.
  • Fake base stations (IMSI catchers): devices are tricked into connecting to a rogue station that requests identifiers or exploits missing checks in early connection stages.

4.2 User Privacy and Location Tracking

  • Identifier re-use: if temporary identifiers (such as GUTI or 5G-GUTI) are not refreshed frequently enough, an attacker can correlate them over time and build movement profiles of a subscriber.
  • Paging analysis: by monitoring paging messages in a cell and combining them with external triggers (such as sending messages or calls), an attacker can infer whether a specific subscriber is currently in a region.

4.3 Core Network and SBA Attacks

  • Exploiting misconfigured APIs: 5G core network functions expose HTTP-based interfaces. If TLS, certificates, or access control are misconfigured, attackers may gain unauthorized access to signaling or subscriber data.
  • Slice isolation failures: incorrect configuration of network slices can allow one tenant or service to affect the performance or security of another, undermining the isolation promised by network slicing.

4.5 Evaluation Criteria

  • Required capabilities: physical location, access to RF spectrum, or access to operator infrastructure.
  • Complexity: required knowledge of radio protocols, cryptography, and implementation details.
  • Detectability: whether unusual signaling patterns, RF behavior, or log entries could reveal the attack.
  • Mitigations: standard-level defenses (e.g., new protocol versions) and deployment-level measures (e.g., monitoring, configuration hardening).

5. Own Opinion and Contribution

In my opinion, the security design of 4G and especially 5G represents a major improvement over previous generations. The introduction of mutual authentication, stronger key derivation, and subscriber identity protection shows that lessons from past failures have been taken seriously. However, the system is now also more complex: the 5G core relies on many different components and interfaces, which increases the attack surface if they are not configured correctly.

From a practical perspective, attacks that require custom RF equipment and detailed protocol knowledge are still mostly limited to motivated adversaries such as security researchers, criminals with technical skills, or state-level actors. On the other hand, misconfigurations of 5G service-based interfaces resemble typical IT security problems and can be exploited with common web security skills. This suggests that mobile network operators need both traditional telecom security expertise and modern IT security practices.

The main contribution of this project is a structured overview that links concrete attacks to the underlying security mechanisms and evaluates their feasibility using simple criteria (required skills, cost, and practicality). This may help students or engineers who are new to the field to quickly understand which threats are mainly of theoretical interest and which ones are realistic in current deployments.

6. Conclusion

This project has described the main security mechanisms in 4G and 5G mobile networks, summarized representative attacks from the literature, and analyzed their feasibility. Compared to earlier generations, 4G and 5G offer much stronger protection of subscriber identity and signaling. Nevertheless, several classes of attacks remain possible, especially in the presence of legacy technologies, protocol downgrades, or weak deployments of the 5G core.

Future work could include hands-on experiments in a controlled testbed, automated tools for detecting misconfigurations in service-based interfaces, and better visualization of the security properties of different network slices. Overall, mobile network security is a moving target, and continuous collaboration between standardization bodies, operators, and the research community is necessary to keep 4G and 5G networks trustworthy.
