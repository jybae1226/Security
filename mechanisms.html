3. Security Architecture & Mechanisms

This page summarizes how 4G and 5G secure subscriber identity, authenticate devices and networks, derive cryptographic keys, and protect signaling and user data.

3.1 4G (LTE) Security

  • Authentication (EPS-AKA): mutual authentication between the User Equipment (UE), the serving base station (eNodeB), and the core network (EPC) using long-term credentials stored in the USIM and in the operator's Home Subscriber Server (HSS).
  • Key hierarchy: a master key K is used to derive KASME and then further keys such as KeNB for the radio interface. This limits the impact of key compromise and supports key refresh during mobility.
  • Confidentiality and integrity: standardized algorithms (for example, AES or SNOW 3G based) provide encryption and integrity protection for control-plane signaling and, optionally, for user-plane data.
  • Separation of planes: different keys and security associations for control and user plane, so that signaling remains protected even if user data is handled by intermediate gateways and tunnels.

3.2 5G Security Enhancements

  • 5G-AKA and subscriber identity protection: the permanent subscriber identifier (SUPI) is never sent in clear over the air. Instead, a concealed identifier (SUCI) is transmitted, protecting users from passive IMSI-catching style attacks.
  • Service-Based Architecture (SBA): core network functions communicate via HTTP/2 and JSON APIs secured with TLS and modern authentication/authorization mechanisms, aligning mobile network security with common IT security practices.
  • Roaming and home control: the home network has more control over authentication decisions and can better enforce its policies even when the subscriber is served by a visited network.
  • Network slicing: logical networks (slices) are isolated and can be configured with different security policies depending on the service (e.g., massive IoT vs. ultra-reliable low-latency communication).
